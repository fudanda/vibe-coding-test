## 为什么改

团队需要让 Codex 和成员在后续开发中同时维护“当前模块事实”和“历史变更原因”，避免模块文档变成流水账，也避免变更缺少上下文。

## 改了什么

- 新增模块开发文档目录 `docs/vibe-coding/modules/`。
- 新增变更记录入口和决策记录入口。
- 新增变更记录模板。
- 在提交与 PR 规范中加入文档同步检查。

## 验证方式

- 静态检查文档文件存在。
- 检查模块索引链接指向真实文件。
- 检查 PR 模板包含文档同步检查项。
