## 为什么改

`vibe-coding-test` 作为测试模板，需要展示计划优先模式的真实执行方式。`update_plan` 不能触发 Codex App 原生计划 UI，所以本地规则需要明确：计划优先模式必须输出待确认方案，并等待用户明确确认后才允许修改代码。

## 改了什么

- 在 `AGENTS.md` 的“模式和计划清单”中补充 Human Loop 确认门禁。
- 明确 `update_plan` 只负责维护计划清单，不等于 Codex App 原生计划模式。
- 明确待确认方案必须包含“状态：等待确认 / 确认后才会修改代码”。
- 明确允许进入实现和继续停留在计划优先模式的用户回复。
