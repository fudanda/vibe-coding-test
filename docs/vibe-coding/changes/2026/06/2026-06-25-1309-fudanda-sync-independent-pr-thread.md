## 为什么改

`vibe-coding-test` 是 Vibe Coding 的测试项目，需要同步通用规则：每次有意义的功能修改完成后，优先在独立 Codex 对话中执行提交与 PR 准备流程，当前开发对话默认不再直接执行 Review 和本地 commit。

## 改了什么

- 更新本项目 `AGENTS.md`，将提交与 PR 准备流程改为默认优先新建独立 Codex 对话。
- 明确当前开发对话只负责实现、验证、文档同步、change fragment 和交付交接包。
- 明确只有用户说“在当前对话直接提交”等授权语时，当前对话才允许执行 Review 和本地 commit。
- 更新 PR 模板中的 Git 提交方式和检查项，区分独立提交准备对话自动提交、当前对话直接提交、人工提交和未提交。

## 模块文档同步

- [x] 不需要更新模块文档，原因：本次只修改项目协作规则和 PR 模板，不改变业务模块当前实现。

## 验证方式

- `rg` 检索确认本项目 `AGENTS.md` 和 PR 模板已包含“独立提交准备对话”“当前对话直接提交”等新口径。
- `rg` 检索确认旧的当前对话默认提交口径已移除。

## 风险和后续事项

- 后续真实功能开发完成后，当前开发对话应停止在交接包，不再直接 commit。
- 如果需要当前对话直接提交，用户必须明确给出“在当前对话直接提交”类授权语。
