# ADR-0001：模块当前事实与历史记录分离

- 日期：2026-06-23
- 状态：accepted
- 决策人：fudanda
- AI 协助：Codex
- 影响范围：文档维护、PR 检查、AI 协作流程

## 背景

团队希望“每次修改都有记录”，但如果把历史过程全部塞进模块文档，模块文档会越来越难读，Codex 后续也更难快速理解当前实现。

## 决策

模块文档只描述当前事实，历史变化放入变更记录，重要取舍放入 ADR。

## 采用方案

- `docs/vibe-coding/modules/*.md` 维护模块当前状态、开发方式和验证方式。
- `docs/vibe-coding/changes/` 保存每个有意义变更的独立日志碎片。
- `docs/vibe-coding/decisions/` 保存重要取舍和放弃方案。

## 放弃方案

- 每次编辑都写进模块文档：会把当前事实和历史流水混在一起，长期维护成本高。
- 只依赖 Git commit：能看到差异，但不够方便团队和 Codex 从文档入口理解业务原因。

## 影响

- 日常 PR 需要判断是否新增 change fragment。
- 模块事实变化时仍要同步更新对应模块文档。
- 重要取舍需要新增 ADR 文件。

## 回看条件

如果团队连续多次忘记补记录，或记录质量不稳定，再考虑增加 PR 自动检查脚本。
